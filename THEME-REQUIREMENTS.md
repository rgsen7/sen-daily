# 森之日报 自定义主题 — 需求框架

> 版本：v0.1 草案
> 日期：2026-02-16
> 状态：需求收集阶段

---

## 设计方向

**一句话定位：** 专业研究机构的信息控制台——功能密集但界面克制，像高端实验室的监控系统，不像大众博客。

**视觉参考：** 21st.dev 组件库风格（shadcn/ui + Tailwind CSS + Radix UI）
- 无装饰主义：没有渐变、没有阴影堆叠、没有圆角过度
- 信息密度高但不拥挤：靠间距和字重区分层级，不靠颜色
- 配色克制：主色不超过 3 个，大面积留白/留黑，强调色只用在关键交互点
- 字体：等宽 + 无衬线混排（代码感 + 可读性）

**风格关键词：** 控制台 / 研究所 / 信息密度 / 克制 / 专业

---

## 配色方案

### 暗色模式（默认）
- 背景：近黑（#0a0a0a ~ #111111），不是纯黑
- 文字：高对比灰白（#e5e5e5），次要信息用中灰（#737373）
- 强调色：冷蓝或青绿（一个就够），只用在链接、标签、活跃状态
- 分割线：极淡（border-white/5 级别），靠间距分区而非线条

### 亮色模式
- 背景：不是纯白，偏暖灰（#fafafa）
- 同样克制，不加多余颜色

## 布局结构

### 首页
- 顶部：站名 + 极简导航（不超过 5 项）
- 主区域：最新日报卡片列表，每张卡片显示日期、标题、6 板块的摘要标签
- 侧边栏（桌面端）：词云 / 快速跳转 / 数据看板入口
- 移动端：侧边栏折叠为底部或汉堡菜单

### 文章页
- 单栏阅读，最大宽度 720px
- 浮动目录（桌面端右侧）
- 6 板块之间用极细分割线或间距区分
- 底部：上一篇/下一篇 + 标签 + 评论

### 归档页
- 时间线视图：按月分组，左侧时间轴，右侧标题列表
- 支持按标签筛选

## 核心功能

### 1. 词云（Tag Cloud）
- 不是传统的彩色词云，是 monospace 风格的标签矩阵
- 字号按频率分级（3-4 级即可），颜色统一，靠字重和大小区分
- 点击跳转到对应标签归档页
- 数据源：文章标签 + 文章内容高频词提取

### 2. 高级搜索
- 支持正则表达式搜索（前端 Fuse.js 升级或自建索引）
- 搜索语法提示：`tag:AI`、`date:2026-02`、`/正则/`
- 搜索结果高亮匹配文本
- 搜索历史记录（localStorage）

### 3. 数据看板（Dashboard）
- 独立页面，展示 AI 行业关键指标
- 数据来源：日报中"数据看板"板块的结构化数据，累积形成时间序列
- 图表类型：折线图（趋势）、柱状图（对比）、数字卡片（最新值）
- 技术方案：Chart.js 或 ECharts（轻量），数据存为 JSON/CSV 在 data/ 目录
- 指标示例：
  - 本周 AI 融资总额
  - 模型发布频率
  - API 价格趋势（GPT/Claude/DeepSeek）
  - 开源模型 Star 增长

### 4. 热点时间线
- 把每条新闻按时间轴可视化
- 横向时间轴或纵向时间线，每个节点是一条新闻
- 点击展开详情
- 支持按公司/人物/事件类型筛选

### 5. 关联图谱（进阶，可后期实现）
- 公司、人物、事件之间的关系网络
- 技术方案：D3.js force-directed graph 或 vis.js
- 数据需要结构化标注（实体识别），可让 Manus 在生成日报时同时输出

### 6. RSS 订阅
- 保留完整 RSS 输出
- 订阅引导放在固定位置但不突兀

### 7. 评论系统
- 保留 giscus（GitHub Discussions）
- 样式适配新主题

## 技术方案

### 方案 A：Hugo 自定义主题（推荐）
- 基于 Hugo 模板系统从零写主题
- CSS：Tailwind CSS（和 21st.dev 同技术栈，风格一致）
- JS：最小化，只在搜索/图表/词云处用
- 优势：完全可控，性能极致（纯静态），部署不变（GitHub Pages）
- 劣势：交互功能受限于静态站

### 方案 B：迁移到 Next.js / Astro
- 如果数据看板和关联图谱需要动态数据，可能需要
- 但目前数据量小，静态 JSON + 前端渲染足够
- 暂不考虑，除非功能需求超出 Hugo 能力

### 结论：方案 A，Hugo + Tailwind CSS 自定义主题

## 21st.dev 风格提取要点（待 Scientist 深度研究补充）

已知的核心风格特征：
1. **shadcn/ui 设计语言**：Radix UI 原语 + Tailwind CSS，组件无装饰、高可组合
2. **CSS 变量驱动主题**：所有颜色通过 CSS 变量定义，暗色/亮色一键切换
3. **Inter 字体**：正文用 Inter，代码用等宽字体
4. **动画克制**：只在必要处用微动画（hover 状态、展开/折叠），不用花哨过渡
5. **信息层级靠排版**：字重（400/500/600）+ 字号 + 间距，不靠颜色区分

等 Scientist 的深度研究结果后补充具体代码样例和设计 token。
